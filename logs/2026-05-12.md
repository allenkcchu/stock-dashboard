## 完成項目

### 修正 FORCE_SEND 表達式（昨日修法無效的根本原因）

**症狀**：今早推播再次未發送，且今天完全無 run 紀錄（或 run 嚴重延遲）。

**根本原因**：昨日 2026-05-11 記錄的修法，其 GitHub Actions 表達式：

```yaml
FORCE_SEND: ${{ github.event_name == 'schedule' && 'true' || github.event.inputs.force }}
```

在 schedule 觸發時，`FORCE_SEND` 實際評估為空字串（非 `'true'`），導致
`is_market_open_window()` 仍然生效，加上 GitHub Actions cron 延遲 2-3 小時，
執行時已是 13:09 ET，不在 9:20–9:59 窗口內，直接 skip。

昨天兩個 scheduled run（UTC 16:13、17:08）都顯示 `success`，
但 log 內均為 `Not in market open window, skipping`，實際未推播。

**修正**：改用 fallback 寫法，schedule 觸發時 `inputs.force` 為空，自動 fallback 到 `'true'`：

```yaml
FORCE_SEND: ${{ github.event.inputs.force || 'true' }}
```
