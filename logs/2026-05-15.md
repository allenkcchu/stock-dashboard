#### fini_net 缺失問題診斷與修補

- 診斷：2024-2026 共 616 筆 fini_net 全為 null
- 根本原因：init 腳本批次跑 ~1250 次 TWSE API，約第 40+ 次後觸發 rate limiting，之後全部回傳 None；因為這些日期已有 TAIFEX 資料所以 row 存在，導致 init 重跑時被 skip
- TWSE API 本身正常（個別測試 5 筆都成功）
- 修法：寫 `scripts/patch_twse_data.py`，只補 fini_net 為 null 的日期，sleep 2s/筆，連續 5 次失敗自動 cooldown 60s
- 結果：560/616 更新成功（剩下 56 筆為台灣國定假日，正常無資料），fini_net 達到 1205/1306（92%）

## 遇到的問題與解法

| 問題 | 解法 |
|------|------|
| `conda run` 不支援多行 `-c` 腳本 | 改寫成 .py 檔，直接呼叫 stock env 的 python.exe |
| `conda run` stdout 遇中文字元 UnicodeError | 改用 `PYTHONUTF8=1 python.exe script.py` |
| 批次 patch 661 筆全失敗（no data） | 2s sleep + 5 次連續失敗觸發 60s cooldown，分三次自動恢復 |
| Python stdout 寫檔時被 buffer | 以 `> log.txt` 重導向時須加 `-u`，或等 script 結束才看檔案 |
